本篇主题：hexo+托管平台，实现0成本建站

首先了解一下hexo:hexo是一个简洁高效、免费的博客框架，这些漂亮的站点也是以hexo为框架的： [Akilarの糖果屋](https://akilar.top/) [安知鱼](https://blog.anheyu.com/) 所以不必担心免费的站点不好看、功能不全。顺便一提，本网站也是以hexo为框架的。

预备工作：安装必要软件

[安装Node.js](https://blog.csdn.net/antma/article/details/86104068)

安装 git

安装成功后分别使用 git --version 和 node --version 命令测试是否安装成功，安装成功后会显示版本号，没有的话就是你安装不对。

安装 hexo: 首先需要建立博客文件夹，建议建在非系统盘，例如 D:/Hexo/(接下来都会以这个目录为例子)，那么这个目录就是我们博客的根目录了。接下来在 hexo 文件夹里右键 - Git Bash Here，复制一下代码粘贴 + enter:

npm config set registry https://registry.npmmirror.com

npm install hexo-cli -g

hexo init

接下来依然在 hexo 文件夹下 git bash, 装插件：

npm install hexo-generator-index --save

npm install hexo-generator-archive --save

npm install hexo-generator-category --save

npm install hexo-generator-tag --save

npm install hexo-server --save

npm install hexo-renderer-marked --save

npm install hexo-renderer-stylus --save

npm install hexo-generator-feed --save

npm install hexo-generator-sitemap --save

npm install hexo-admin --save

npm install hexo-deployer-git --save

到这里基本已经好了。接下来学习一下基本指令：

hexo clean—— 清除缓存

hexo g—— 生成静态网页文件

hexo s—— 本地预览

hexo d—— 部署到云端

hexo new “111”—— 新建一篇名字为 111 的文章

hexo new page xxxxx—— 新建一个名为 xxxxx 的页面

接下来你需要自学 Markdown 语法用来写博客，语法不难（至少比 html 和 css 简单）

部署到某些平台

有很多平台，比如gitee(速度快，实名认证，免费分配域名)、github（速度慢，不用实名，免费分配域名）Vercel（速度较快，不用实名，免费分配域名）Netlify（速度较快，不用实名，免费分配域名）

我自己的建议，首先 gitee 是实名认证的，速度很快（估计是四个里边最快的），但审核要更严（如果你想做些奇怪的事就别用这个）github 国内总访问不开，不过你可以配上 cloudflare。我推荐 netlify 和 vercel。这两个比较快而且安全。
